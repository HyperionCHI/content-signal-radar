# Podcast Signal Extraction Prompt

把播客内容压缩成对 Maple 真有用的信号，不是节目简介，也不是逐段复述。

Maple 关注的是：

- AI Agent / AI 产品 / LLM 工作流
    
- 独立开发 / 出海 / 增长 / 商业化
    
- 产品判断、MVP、用户需求
    
- 技术趋势、AI 工程、开源模型、开发工具
    
- 个人品牌、内容策略、创作者工作流
    
- 个人成长系统、知识管理、语言学习方法
    

## 任务

从一期播客或视频里提炼：

1. 最值得记住的判断
    
2. 最值得转成内容的表达
    
3. 最值得迁移到产品、增长、技术或个人系统里的方法
    
4. 最值得 Maple 今天行动的点
    

不要复述对谈过程。

如果这一期只是闲聊、宣传、信息密度低，直接输出：

无高信号内容。

## 输出格式

### 核心判断

开头一句：

这一期最值得 Maple 记住的是：……

这句话必须是判断，不是主题介绍。

不要写：

这期节目讨论了 AI 和创业。

要写：

这期真正有价值的判断是：独立开发者做 AI 产品，壁垒不在模型，而在分发入口和垂直工作流。

### 信号等级

给出一个等级：

- S：必须进入总简报
    
- A：值得进入总简报
    
- B：可作为备选信号
    
- C：低信号，不建议进入主输出
    

说明评分原因。

### 信号类型

从下面选择 1-3 个：

- market_signal
    
- product_signal
    
- growth_signal
    
- tech_signal
    
- content_signal
    
- system_signal
    
- language_signal
    

### 3-5 个高信号点

每个点都按以下结构输出：

#### 信号点：[标题]

- 核心观点：他说了什么
    
- 为什么重要：这为什么不是普通聊天
    
- Maple 可以怎么用：对内容、产品、增长、技术、个人系统或语言学习有什么用
    
- 可执行动作：Maple 今天可以做什么
    
- 风险判断：这条观点可能哪里不适用
    
- 原始链接：保留原始视频链接
    

## 特别要求

如果原文里有特别锋利、适合引用的一句话，可以引用 1 句。

引用必须满足：

- 只引用 1 句
    
- 不要长引用
    
- 不要为了引用而引用
    
- 必须带原始链接
    

如果没有值得引用的原话，不要硬引。

## 内容转化判断

如果这一期适合转成内容，补充：

### 可写角度

- 角度标题
    
- 适合平台：X / 即刻 / 长文 / Newsletter
    
- 一句开头
    
- 为什么能写
    

如果不适合，写：

不适合转成内容，原因是……

## 产品 / 项目判断

判断这一期是否能启发 Maple 当前或未来项目：

- AI Agent 工具
    
- 翻译 / 多语言学习工具
    
- 简历分析工具
    
- 跳舞 / SwingDict 工具
    
- 个人内容系统
    
- 独立开发产品组合
    
- Vibe Coding 课程 / 方法论
    

输出：

- 可能影响的项目
    
- 具体启发
    
- 是否值得验证
    
- 最小验证动作
    

如果没有明显关系，写：

暂无直接项目影响。

## 写作要求

- 默认中文
    
- 直接、清楚、少废话
    
- 不要写成节目简介
    
- 不要做嘉宾背景介绍，除非背景本身影响判断
    
- 不要复述对话流程
    
- 不要输出没有行动价值的观点
    
- 不要编造
    
- 如果信息不足，直接说信息不足，不能判断
    
- 必须带上原始视频链接
