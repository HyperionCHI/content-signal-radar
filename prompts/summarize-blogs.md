# Blog Signal Extraction Prompt

把 blog 文章变成 Maple 可用的决策信号，不是新闻摘要，也不是文章复述。

Maple 关注的是：

- AI Agent / AI 产品 / LLM 工作流
    
- 独立开发 / 出海产品 / Micro SaaS
    
- 增长、分发、SEO、冷启动、商业化
    
- 用户需求、产品机会、MVP 假设
    
- 技术底层变化、工程实践、开源模型
    
- 个人品牌、内容策略、知识管理
    
- 英语 / 日语 / 多语言学习方法
    

## 任务

阅读输入文章，判断它是否包含对 Maple 有价值的信号。

不要问“这篇文章讲了什么”，要问：

1. 它证明了什么变化？
    
2. 它暴露了什么需求？
    
3. 它影响了什么产品判断？
    
4. 它能不能变成一个内容选题？
    
5. 它能不能启发一个 MVP？
    
6. 它能不能改变 Maple 的工作流、技术栈或学习系统？
    

如果答案都是否定的，直接输出：

无高信号内容。

## 优先提取

优先提取以下类型：

- 新产品发布、功能变化、定价变化、商业模式变化
    
- AI Agent、LLM、AI Coding、RAG、开源模型相关变化
    
- 对独立开发者有用的工程实践、增长策略、分发方法
    
- 用户痛点、市场需求、可验证的 MVP 场景
    
- 能转成 X / 即刻 / 长文的观点
    
- 对个人工作流、知识管理、语言学习有实际帮助的方法
    

## 降权内容

以下内容不要重点输出：

- 纯宣传
    
- 普通融资新闻
    
- 没有具体细节的产品公告
    
- 大而空的行业判断
    
- 没有用户、场景、动作的观点
    
- 无法迁移到 Maple 的产品、内容、技术或个人系统的内容
    

## 输出格式

### 核心判断

用 1 句话说明：

这篇文章真正宣布了什么 / 证明了什么 / 暴露了什么？

### 信号等级

给出一个等级：

- S：必须进入总简报，会影响 Maple 的产品、内容或技术判断
    
- A：值得进入总简报，有明确启发
    
- B：可作为备选信号，有一定参考价值
    
- C：低信号，不建议进入主输出
    

同时说明评分原因。

### 信号类型

从下面选择 1-3 个：

- market_signal
    
- product_signal
    
- growth_signal
    
- tech_signal
    
- content_signal
    
- system_signal
    
- language_signal
    

### 关键事实

列出文章中的关键事实。

如果有以下内容，必须点出来：

- 具体数字
    
- 基准测试
    
- 功能变化
    
- 定价变化
    
- 用户量 / 收入 / 增长数据
    
- 模型性能
    
- 技术架构
    
- 产品流程变化
    
- 真实用户反馈
    

不要编造文章里没有的数字。

### 为什么现在值得 Maple 关心

解释这件事对 Maple 的意义。

必须落到至少一个方向：

- 内容选题
    
- 产品判断
    
- MVP 机会
    
- 增长 / 分发
    
- 技术栈 / 工作流
    
- 个人系统
    
- 语言学习
    

### 可迁移启发

说明 Maple 可以怎么用。

可以是：

- 一个内容角度
    
- 一个产品假设
    
- 一个功能设计
    
- 一个增长动作
    
- 一个技术测试
    
- 一个个人系统实验
    
- 一个语言学习实验
    

### 风险 / 误区

指出这篇文章可能被误读的地方。

例如：

- 看起来像趋势，其实只是单一案例
    
- 看起来适合独立开发，其实冷启动很难
    
- 看起来是技术突破，其实还没有产品化价值
    
- 看起来是增长方法，其实不可复制
    
- 看起来适合做产品，其实没有明确付费场景
    

### X / 即刻角度

如果适合转成内容，给出：

- 角度标题
    
- 一句开头
    
- 为什么能写
    

如果不适合，写：

不适合转成内容，原因是……

### 对 Maple 项目的影响

判断是否能启发 Maple 当前或未来项目：

- AI Agent 工具
    
- 翻译 / 多语言学习工具
    
- 简历分析工具
    
- 跳舞 / SwingDict 工具
    
- 个人内容系统
    
- 独立开发产品组合
    
- Vibe Coding 课程 / 方法论
    

如果没有明显关系，写：

暂无直接项目影响。

### 原文链接

必须保留原文链接。

## 写作要求

- 默认中文
    
- 直接、清楚、少废话
    
- 不要写成新闻摘要
    
- 不要说空话
    
- 不要脑补
    
- 不要为了显得有用而硬找意义
    
- 如果信息不足，直接说信息不足，不能判断
